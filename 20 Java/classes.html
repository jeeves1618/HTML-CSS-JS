  • Static Variables and Methods

    Static variables and methods are properties of the class. So, every instance of the class shares the same static variables. So, if changes are made to the static variables, they will be reflected across all instances. The static variables can be accessed through object reference variable as well. But, it is considered a best practice to access them through class name. Here is a bad way of accessing them..

    Static methods cannot access the instance variables and methods directly. We will get an error non-static variable this cannot be referenced from a static context. So, static methods are used only in the scenarios where they do not need any data from 'this' instance.

  • POJO

    A plain old java object (or POJO) is a class that generally has instance fields and their getters and setters only. It does not have any other business functionality built within. POJOs are used to pass data between classes which house such business functionality.

    Many data base frameworks (like Spring JPA) uses POJO to read data from or write data to databases, files or streams. POJO used for interfacing with databases are called entities and may be annotated as such. It may also be called a bean or a Java bean. An example for simple POJO invocation is here.

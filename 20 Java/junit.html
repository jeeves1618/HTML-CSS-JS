JUnit

  • JUnit Basics

    Get the JUnit Jupiter dependency from mvn repository Then you can write tests like these
    @Test
    void equalityTest(){
    DemoUtils demoUtils = new DemoUtils();
    assertEquals(6, demoUtils.add(2,4),"The add function is not working properly");
    assertNotEquals(7,demoUtils.add(2,4),"The add function should not return this output");
    }
     @Test annotation will get the code tested. Here the instantiation of DemoUtils is something that may have to be done before each test. In that case you can bundle that up in a separate method called lifecycle method.

    There are 4 such methods: @BeforeEach , @AfterEach, @BeforeAll and @AfterAll

  • Lifecycle Methods

    We can introduce the code to be executed before or after a test method through the annotations @BeforeEach and @AfterEach. Here is how the code looks.
    DemoUtils demoUtils;
    @BeforeEach
    void createObject(){
        demoUtils = new DemoUtils();
    }
    @Test
    void equalityTest(){
        assertEquals(6, demoUtils.add(2,4),"The add function is not working properly");
        assertNotEquals(7,demoUtils.add(2,4),"The add function should not return this output");
    }
    @BeforeAll and @AfterAll will get executed only once before all tests and after all tests respectively. These two methods must be static.

  • Display Names

    We can give a display names for the test functions.
    @Test
    @DisplayName("JIRA1234 - Acceptance Criteria 7")
    void equalityTest(){
        DemoUtils demoUtils = new DemoUtils();
        assertEquals(6, demoUtils.add(2,4),"Not working properly");
        assertNotEquals(7,demoUtils.add(2,4),"Should not return this");
    }

    We can also have the names generated. @DisplayNameGeneration (DisplayNameGenerator.ReplaceUnderscores.class) will generate a name by replacing underscores. There are few other options while generating names.

  • Few other assertions

    assertTrue() will check if the returned Boolean Value is true. assertFalse() will check for the opposite.

    assertSame() will check if the returned addresses refer to the same Object. assertNotSame() will check for the opposite.

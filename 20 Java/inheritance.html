Inheritance

  • super()

    super() is a way to call a constructor on the super class, directly from the sub class's constructor. Like this(), it has to be the first statement of the constructor. Because of this rule, super() and this() can never be called from the same constructor.

    So, it is a challenge if we want to manipulate data passed as parameter before calling super. We can try something like this.

    super(isTurboCharged,engineVolumeInCC < 1000 ? "Small" : (engineVolumeInCC > 1500 ? "Large" : "Medium"))

    If a subclass doesn't make a call to super(), java will make that call to the default constructor of the super class. If the default constructor of the super class isn't there, because a constructor was explicitly defined, then the subclass must explicitly call that constructor in the super class.

    A object of type sub class can be passed to the method accepting the super class. But, the converse is not possible.

  • Composition

    Inheritance defines an IS A relationship. Composition defines an HAS A relationship.

    We started talking about composition in the previous section itself to compare it against inheritance. Inheritance is a way to reuse functionality and attributes. Composition is a way to make the combination of classes, act like a single coherent object.

    As a general rule, when you are designing programs in Java, you probably want to look at composition first. In the personal computer example, the Main module didn't have to know anything about the parts, to access the personal computer to do something.

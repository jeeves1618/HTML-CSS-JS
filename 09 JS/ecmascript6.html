ES6 and Beyond

  • Let and Const

    Prior to ES6, we were declaring variables using var. It is scoped by function. For instance if you declare a variable using var inside a for loop within a function, that variable will be accessible ouside the loop within that function.

    ES6 streamlined it better with the key words let and const to assign variables scoped by block. So, if you have a variable defined within an if block, it will not available outside of it.

    So, when to use what? There is no right or wrong here. But, based on my readings, I have these thumb rules.

    1. Use const by default.
    2. If you have to reassign values to that variable, which are also shared across many scopes, you may want to use var. There are people who say var shouldn't be used in ES6.
    3. For smaller scopes with reassignments, always use let

  • Array Destructuring

    Destructuring is an efficient way to extract multiple values from the data that is stored in arrays or objects. In case of arrays, destructuring basically helps us to get the elements without iterating over the array. For instance,
    let team83 = [ "Kapil Dev", "Mohinder Amarnath", "Kirti Azad"];
    let [p,q,r] = team83;
    p, q and r will now have "Kapil Dev", "Mohinder Amarnath" and "Kirti Azad" assigned respectively. Array destructuring is based on the position of the elements. Here is an example of array destructuring..

  • Object Destructuring

    Object destructuring is based on the name of the properties. You can use the name of the properties to destructure and get the values. Here is an example
    let sunseeker55evo = {
    engine: "TWIN VOLVO PENTA D11-725 (2 X 725PS)",
    performance: "32 KNOTS",};

    let { engine, performance } = sunseeker55evo; will store the values from the object properties to the variables defined within curly braces. If you want the variable names to be different from the property names, you can use let { engine: motor, performance: topSpeed } = sunseeker55evo;. Now, motor and top speed will have the values of the engine and performance properties. An example of Objec Destructuring is available here.

  • Arrow functions in ES6

    Arrow function needs a return only if it has more than one line of code. Otherwise, it can be super simple as below.

    1. let cubed = num => {
    2. console.log(num*num*num);
    3. };
    4. cubed(3);//Calling the arrow function

    If you want to pass multiple parameters, then the parameters should be enclosed within parentheses.

    1. let cubed = (num1,num2,num3) => {
    2. console.log(num1*num2*num3); };
    3. cubed(3,4,5);//Calling the arrow function

    Arrow functions are used primarily for small pieces of repeatable code. If the only job of the arrow function is to return an value, then you can simplify it further as below.

    1. let areaOfCircle = radius => 22/7*radius*radius;
    2. aresOfCircle(10); //Calling the above function.

  • Template Literals

    Without template literals, when you want to combine output from expressions with strings, you'd concatenate them using the addition operator +:

    1. const a = 5;
    2. const b = 10;
    3. console.log("Fifteen is " + (a + b) + " and\nnot " + (2 * a + b) + ".");

    With template literals, you can avoid the concatenation operator — and improve the readability of your code — by using placeholders of the form ${expression} to perform substitutions for embedded expressions.

    1. const a = 5;
    2. const b = 10;
    3. console.log(`Fifteen is ${a + b} and not ${2 * a + b}.`);

    Template literals coerce their expressions directly to strings, while addition coerces its operands to primitives first.

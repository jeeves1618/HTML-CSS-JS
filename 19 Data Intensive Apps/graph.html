Graph Database

  • Graph Tables

    Relational data models primarily serve the purpose of storing data with one-to-one, one-to-many and simple many-to-many relationships. The one-to-one and one-to-many relationships are also well managed by document databases like MongoDB.

    But, what if complex many-to-many relationships are common in our data? As the connections between our data gets sophisticated, it becomes natural to model the data as graph.

  • Nodes and Relationships

    A graph consists of two different kinds of objects: nodes (or vertices or entities) and relationships (or edges or arcs). Similar type of nodes are clubbed together and are assigned a label. So, you can think of nodes as records in a RDMS table and label as the table.

    Unlike RDMS, where relationships are defined at the table level, the relationships in Graph DB can be defined at the node level.

  • Properties

    We can assign the properties to nodes and relationships. A relationship will have a direction and a label. The label will say something lile 'belongs to', 'travelling to' etc.

    For instance, we can have a node with properties {name: "John", ...} or {name: "Peter", ...}. Those nodes can have multiple labels, like citizens, employees etc. But, relationships can have only one label.

  • Neo4j

    Neo4j is the most popular enterprise strength graph database in use. It is schema on read and the properties of nodes and relationships can differ under similar label.

    Neo4j does not allow nested properties for node and relationship. The easiest way to try creating and accessing Graph DBs is to install Neo4j Desktop by going to neo4j.com -> Developers -> Download Center.

  • Neo4j Browser

    Once the neo4j desktop is installed, create a new project, a new DBMS and start the DBMS. The DBMS will be running in the local and will be exposes in the port 7474. You can click on open button next to the DBMS. It will have an option to open the Neo4j browser.

    So, now you have two applications running on your machine, Neo4j Desktop and Neo4j Browser. You can also open the Neo4j browser through http://localhost:7474/. create database will create the database under the DBMS created before.:use database-name will switch to database-name.

  • Cypher QL

    The query language used in the Neo4j is the Cypher (after matrix character) Query Language or CQL. drop database-name is the CQL for dropping database-name.

    CQL refers to node with parantheses. So, create() will create a node without any label or properties.create(:Author) will create a node with the label Author. create(:Author:Reader) will create a node with two labels Author and Reader. If you want the created node to be returned right away, use an alias and return the alias as in create(aliasrt:Author:Reader) return(aliasrt)

  • Match Statement

    match (abc) return abc will retrieve all the nodes and will display them as graph. abc is just an alias. The nodes will have default ids assigned.

    The above query is a SELECT ALL in RDBMS parlance. We will see how we can write efficient matches below.

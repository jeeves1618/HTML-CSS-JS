Basics of HTML and CSS

HTML Notes

HTML is always redendered from top to bottom.

Prior to HTML5, all content models are either block level elements or inline elements. The content models in HTML 5 are documented here. https://www.w3.org/TR/2011/WD-html5-20110525/content-models.html

HTML is only for structuring a page. CSS can be used to style h1 as h2. So why should we use them? Wny can't we simply use <div> or <p> element then? Because, using disticnt semantic elements enhances the readability and provides a structure to the html document. It is for the meaningful expression of the structure of a page.

HTML entities help avoid rendering issues.

We have three types of selectors, element selectors, class selectors and id selectors.

Basics of CSS

CSS declaration block contains one or more declarations. A declaration is always formatted as a property followed by a colon and then a value followed by a semicolon. The colon and semicolon can be followed by zero or more spaces. In nearly all cases, a value is either a single keyword or a space-separated list of one or more keywords that are permitted for that property. If you use an incorrect property or value in a declaration, the whole rule will be ignored.

Selectors in CSS

Selectors can be grouped.h2, p {color: gray;} By placing the h2 and p selectors on the left side of the rule and separating them with a comma, you’ve defined a rule where the style on the right (color: gray;) applies to the elements referenced by both selectors. The comma tells the browser that there are two different selectors involved in the rule. Leaving out the comma would give the rule a completely different meaning, which we’ll explore in “Descendant Selectors”

If you want the attributes to be applied to multiple selectors at a time, then you have to specify the selectors with comma in between. For example, p,h1,.classname {...}

Element with Class Selector you want the attributes to be applied to a select combination of two selectors at a time, then you have to specify the selectors with dot in between. For example p.classname {...} will be applicable to <p class="classname"> but not to <div class="classname">. The same logic can be used on a combination of two class names as well. For instance, .classname1.classname2 {color:blue;} will color the element with both classes, <div class="classname1 classname2"> , as blue

Child Selector If you want the attributes to be applied to a selector (lets say p) only when it is under the block (or a direct child) article, then you have to specify them as article > p {...}

Descendant Selector What if you want to go beyond direct child into grand kids? If you want the attributes to be applied to p element under all descendants within article, then you have to specify them as article p {...}

Multiple Classes If you want to specify two classes for a HTML element, you can just list them under class attribute separated by spaces <div class="classone classtwo">

The child and descendant selectors are not limited to HTML elements. They can be implemented on classes as well. For instance .colorBlue p { color: blue;} CSS tag will have every p inside (at any level) the element with class="colorBlue" colored blue. Similarly, div > .colorBlue{ color: blue;} will have the elements with class="colorBlue" which are direct children of div element colored in blue.

Things we aren't covering here are Adjacent Sibling Selector (Selector + Selector) and General Sibling Selector (Selector ~ Selector).

CSS Psuedo Class Selectors

A CSS pseudo-class is a keyword added to a selector that specifies a special state of the selected element(s). For example, :hover can be used to change a button's color when the user's pointer hovers over it. The syntax of pseudo-classes:

selector:pseudo-class { property: value; } You can also have multiple pseudo classes like
selector:pseudo-class1:pseudo-class2 { property: value; } For example, div:nth-child(3):hover{ background-color: green; cursor: pointer; } is valid.

Here we are going to cover the following pseudo-class selectors.

  • :link
  • :visited
  • :hover
  • :active
  • :nth-child

CSS Style Placement

  1. In line styling. You can put the style element inside the tag of the HTML element. For example <p style="text-align: center;"> is valid.
  2. Styling in head. You can put the same style element inside the head of the HTML document using style tag. For example <head>
    <p <style>text-align: center;</style>
    </head>     is valid.
  3. External Style sheet. <link rel="stylesheet" href="filename.css">

CSS Rules Conflict Resolution

  1. Origin Precedence (when there is conflict). HTML is processed sequentially from top to bottom. So, the CSS rules will also be applied sequentially in the same way and the last declaration always gets reflected. For the same of precedence, think of external CSS as expanded at the point where it was declared inside the head.
  2. Origin Precedence (when there is no conflict). When there is no conflict on the properties, i.e. different properties are assigned values at multiple levels, they merge together
  3. Levels of specificity First is inline styling within HTML tag with a style = " ". Next comes ID, then comes class, psuedo class and attribute. Finally the number of HTML elements. Specificity score is calculated from First to last. It will 1 0 0 0 for a style element with inline styling and 0 1 1 0 for an element with ID and class. Since 1000 > 110, the first will have precedence over the second.
  4. Skipping specificity If you want to override all rules, use !importat after the value of the attribute. p { color:green !important; }

Text Styling

  1. Font Family. Specifies the font collections.
  2. Color Text color
  3. Font Style Font styles is where you are going specify italic etc.
  4. Font Size Specify the size of font in pixels. We can state the size in %, for example font-size: 120%. We can also say font-size: 2em which will make the text twice as larger. em will have a cumulative effect. It will not override. For instance, if you specify the text as 2em in the body and 4em in the div, the text in div will become 8em.
  5. Font Transform Transforms the text to upper case etc.
  6. Text Align Used to align the text to right, left or center

Box Model

© sabe.io

  1. Margin. Every browser will have a default margin. If you want to reset the marging, you can add style element to body as below.
    body { margin: 0; padding: 0; }
    This proper will stop applying to child elements once you override margin in any one of the child elements. We will have to use the * element to enforce the inheritance then.
  2. Creating a box with margin, border and padding.
    #box { padding: 10px 10px 10px 10px; border: 20px solid black; margin: 40px; width 300px; height:auto } The width here does not include the width of padding, margin and border. If you want the width to be inclusive of all three, use box-sizing: border-box; property.
    Now, if you define a height for this box and this defined height happened to be not big enough to hold the text inside, then the text will overflow. If you want the text to be shown within the box and enable a scroll bar, use the property {overflow: auto;}
  3. Inheritance of properties. Some CSS properties are inherited from parent element (e.g body) to child element (e.g div). box-sizing: border-box; is not one of those properties. If you want such properties to be applied to all elements under body, then you have to code like this * {box-sizing: border-box;} using universal selector
  4. Cumulative and Collapsing Margins. If there are two elements side by side and the left element has the right margin of 30 and the right element has the left margin of 20, then the margin between them will be 50 (cumulative). But, if there are two elements one above the other and the top element has the bottom margin of 30 and the bottom element has the top margin of 20, then the margin between them will be 30 (Collapsing).

The Background Property

  1. Creating background for a div.
    #bg {
    width 300px;
    height:500px; background-color: blue;
    background-image: url("a.jpg");
    background-repeat: no-repeat;
    background-position: top right;
    }
    You can also combine all 'background-' properties into one background property as #bg {
    width 300px;
    height:500px; background: blue url("a.jpg") no-repeat top right;
    }     .
    Now, if you define a height for this box and this defined height happened to be not big enough to hold the text inside, then the text will overflow. If you want the text to be shown within the box and enable a scroll bar, use the property {overflow: auto;}
  2. background will override all previously defined background- properties. If a background- property is not explicitly defined in background, then the browser default will override what was specified in background-

Floating, Relative and Absolute

This is the content that comes after floating boxes

Floating an element takes it out of the document flow. Supposing that p1 is inside div01, the div01 will collapse without taking the p1 into consideration. It will be as high as the p2, p3, p4 and section.
#div01 { background-color: #00ffff; }
p[float] { width: 50px; height: 50px; border: 1px solid black; float: right; }
#p1 { background-color: bisque; }
#p2 { background-color: rgb(190, 162, 127); }
#p3 { background-color: rgb(168, 189, 168); }
#p4 { background-color: rgb(150, 182, 204); }
Clearing the float If you want an element that is defined as float to not float around and take its position in the document, then use {clear : left;}

Absolute Positioning: Absolute positioning will place the element relative to the parent/grand parent element which is defined as position: relative. If there are no elements in the ancestory defined as position: relative, then it will place its relative to the html element.

Relative Positioning: Relative positioning offsets the element relative to its normal document flow position.

Media Queries

The syntax of the media query is as below:

@media
(max-width: 767px) {
p {
color: blue;
}
}
The style inside the outer curly braces will be applicable if the condition after @media resolves to true. In this case, (max-width: 767px) should resolve to true for the paragraph to become blue.
Some common conditions in media query are
@media (max-width: 500px) {...}
@media (max-width: 500px) and (max-width: 900px) {...}
@media (max-width: 500px) , (max-width: 900px) {...} //Comma is equal to OR
@media (min-width: 600px) {...}
@media (orientation: potrait) {...}
@media screen {...}
@media print {...}

Active Directory

  • Regions and Services

    Microsoft has built lot of data centers for Azure. These data centers are called regions. The available Azure regions are documented here. You can pick the region closest to you.

    Some regions can have more than one physical data center enabling greater availability for the users. When a region has more than one data center, it is said to have availability zones.

    Azure has also set up complete fall backs for some regions. That is, when one full region fails, the other one can fill its place. These paired regions are set up by Microsoft and cannot be changed by us.

    Here is the list of Azure services. Please click on the categories to explore more.

  • Azure Portal

    Creation of account requires credit card. It will not be charged. Once we have the account created, we will have access to https://portal.azure.com/. Azure portal is the main dasboard from which we can access all its services and resources.

    A resource group is a container that holds related resources for an Azure solution. The resource group can include all the resources for the solution, or only those resources that you want to manage as a group. Search for resource groups and create a new one attached to the subscription of our choice. We should also select the closest location. The tags are optional. We can search for this resource using its name.

    We can also use the CLI option in the portal. Using Azure Cloud Shell will require Azure file share to persist. The Azure file share will be created under a new resource group within the selected subscription.

  • CLI Commands

    Azure has two interfaces for CLI commands, Azure Bash and PowerShell. All Bash CLI comments start with az. az group create --location southindia -n cliTest-rg will create a new resource group called cliTest-rg in the location southindia.

    PowerShell accomplishes the same through a different syntax. New-AzResourceGroup -Name PSTest-rg -Location southindia will create a new resource group called PSTest-rg in the location southindia.

    Azure Bash can be integrated into other applications like Python while PowerShell will not lend itself to such integrations.

  • Resource Groups

    Resource groups are nothing but a logical container for resources. Resource groups are completely free. We can create as many as we want. Usually, we create separate resource groups for Development, Test and Production. We will also have separate resource groups for applications.

    A subscription is also a logical container. So, what is the difference between a subscription and a resource group? Azure provides four levels of management: management groups, subscriptions, resource groups, and resources. This link has the diagram that shows the relationship between these levels. Resource Groups are usually prefixed or suffixed by RG or rg.

  • Storage Account

    Storage accounts are special type of accounts used to store anything in Azure. They are used to store implicit data like VM disks, database backups and other diagnostic data.

    It can also be used for storing application data. Storage account is very cheap and can be used to store GBs of data.

  • Selecting Regions

    The major factor behind the selection of region is the proximity to the application users. Another factor is the availability of required services and products in the regions.

    If you have strict redundancy requirements, you may also want to choose a region with availability zones. Another important factor is pricing. Pricing is not same across all regions.

  • Budgeting

    Almost everything in cloud costs money. There are few pricing models we need to be aware. One is pricing per resource. For instance, we can have a VM and pay for that VM irrespective of usage. We can also pay per consumption. Another way of paying is reserving a resource for a specified duration and paying upfront.

    Always check the resource's cost before provisioning. You can look for const effective alternatives like reservations first. We can check the cost of services using the pricing calculator.

    We can also set up the billing scope to ensure that we are within budget. Search for "Cost Management + Billing" in the portal. Under cost management, we should select budgets and create a new budget, threshold and notification.
